## 1. 상품 목록 조회
### [유저 스토리]
- 사용자는 여러 상품을 확인할 수 있다.
  - 사용자는 각 상품의 브랜드명, 상품명, 상품 가격을 확인할 수 있다.
  - 로그인한 사용자는 각 상품의 좋아요 여부를 확인할 수 있다.
- 사용자는 최신순, 가격 낮은 순, 좋아요 많은 순으로 상품을 정렬할 수 있다.
- 사용자는 브랜드별로 상품을 필터링할 수 있다.
- 사용자는 페이지 번호를 선택해 페이지별 상품 목록을 확인할 수 있다.

### [기능 흐름]
1. (선택) 정렬 기준 제공
2. (선택) 특정 브랜드 필터링 값 제공
3. (선택) 페이징 정보(페이지 번호, 페이지당 상품 수) 제공
4. 각 조건에 맞는 상품 목록 조회
5. 각 상품의 기본 정보(브랜드명, 상품명, 상품 가격)와 좋아요 수 반환
6. 로그인한 사용자일 경우, 각 상품의 좋아요 여부도 포함하여 반환

### [제약 조건]
- 로그인하지 않은 사용자일 경우, 모든 상품이 '좋아요'하지 않은 상태로 조회된다.
- 정렬 기준을 지정하지 않은 경우, 기본 정렬(최신순)을 적용한 상품 목록이 조회된다.
- 브랜드 필터링 값을 지정하지 않은 경우, 빈 상품 목록이 조회된다.
- 페이지 번호를 지정하지 않은 경우, 첫 번째 페이지 상품 목록이 조회된다.

### [오류 처리]
- 유효하지 않은 정렬 기준으로 요청한 경우, 400 Bad Request 응답
- 존재하지 않은 브랜드로 필터링을 요청한 경우, 404 Not Found 응답
- 유효하지 않은 페이지 번호로 요청한 경우, 400 Bad Request 응답
- 시스템 오류가 발생한 경우, 500 Internal Server Error 응답

## 2. 상품 상세 조회
### [유저 스토리]
- 사용자는 특정 상품의 상세 정보를 확인할 수 있다.
  - 상품명, 브랜드명, 재고, 상품 가격, 좋아요 수, 상품 등록 날짜를 확인할 수 있다.
- 로그인한 사용자가 해당 상품을 '좋아요'했는지 여부를 확인할 수 있다.

### [기능 흐름]
1. 특정 상품의 상세 정보 조회
2. 상품이 존재하는 경우, 상품의 상세 정보 반환
3. 로그인한 사용자일 경우, 상품의 좋아요 여부도 포함하여 반환

### [제약 조건]
- 로그인하지 않은 사용자일 경우, '좋아요'하지 않은 상태로 조회된다.

### [오류 처리]
- 존재하지 않는 상품을 요청한 경우, 404 Not Found 응답
- 상품 ID 형식이 잘못된 경우, 400 Bad Request 응답
- 시스템 오류가 발생한 경우, 500 Internal Server Error 응답

## 4. 상품 좋아요 등록
### [유저 스토리]
- 로그인한 사용자는 상품에 좋아요를 누를 수 있다.
- 사용자는 동일 상품에 중복으로 좋아요 등록을 할 수 없다.

### [기능 흐름]
1. 로그인한 사용자가 특정 상품에 좋아요 등록 요청
2. 사용자의 해당 상품 좋아요 내역, 좋아요 등록 일시 저장
3. 해당 상품의 좋아요 수 1 증가
4. 성공 시 좋아요 여부(좋아요됨)와 현재 좋아요 수 반환

### [제약 조건]
- 로그인하지 않은 사용자는 좋아요 등록을 할 수 없다.
- 이미 좋아요한 상품에 다시 좋아요 등록 요청을 보낼 수 없다.

### [오류 처리]
- 로그인하지 않은 사용자가 요청한 경우, 401 Unauthorized 응답
- 존재하지 않는 상품에 대해 좋아요 등록 요청한 경우, 404 Not Found 응답
- 상품 ID 형식이 잘못된 경우, 400 Bad Request 응답
- 이미 좋아요한 상품에 대해 중복 요청을 한 경우, 400 Bad Request 응답
- 시스템 오류가 발생한 경우, 500 Internal Server Error 응답

## 6. 좋아요한 상품 목록 조회
### [유저 스토리]
- 로그인한 사용자는 자신이 좋아요한 상품 목록을 확인할 수 있다.
  - 사용자는 각 상품의 브랜드명, 상품명, 상품 가격을 확인할 수 있다.
- 사용자는 자신이 좋아요한 상품을 좋아요 등록 일시 기준 최근 순으로 확인할 수 있다. 
- 사용자는 페이지 번호를 선택해 페이지별 좋아요한 상품 목록을 확인할 수 있다.

### [기능 흐름]
1. (선택) 페이징 정보(페이지 번호, 페이지당 상품 수) 제공
2. 로그인한 사용자가 좋아요한 상품 목록 조회
3. 각 상품의 기본 정보(브랜드명, 상품명, 상품 가격) 반환

### [제약 조건]
- 로그인하지 않은 사용자는 좋아요한 상품 목록을 조회할 수 없다.
- 페이지 번호를 지정하지 않은 경우, 첫 번째 페이지 좋아요한 상품 목록이 조회된다.
- 좋아요한 상품이 없는 경우, 빈 목록이 반환된다.
- 정렬 기준은 "좋아요 등록 일시 기준 최근 순"으로 고정되고, 다른 정렬 기준을 지원하지 않는다.

### [오류 처리]
- 로그인하지 않은 사용자가 요청할 경우, 401 Unauthorized 응답
- 유효하지 않은 페이지 번호로 요청한 경우, 400 Bad Request 응답
- 시스템 오류가 발생한 경우, 500 Internal Server Error 응답

## 7. 주문 요청
### [유저 스토리]
- 로그인한 사용자는 여러 상품을 한 번에 주문할 수 있다. 
- 사용자는 주문할 상품과 수량을 지정할 수 있다.
- 사용자는 배송 정보를 입력할 수 있다. 
  - 수령인 이름, 수령인 전화번호
  - 기본 주소, 상세 주소
- 사용자는 주문 완료 후 주문 내역을 확인할 수 있다. 

### [기능 흐름]
1. 로그인한 사용자가 주문할 상품과 수량, 배송 정보를 지정하여 주문 요청
2. 각 상품의 재고 확인
3. 사용자의 포인트 잔액 확인
4. 모든 상품의 재고가 충분하고, 사용자의 포인트가 총 주문 금액 이상인 경우:
   - 각 상품의 재고 차감
   - 사용자의 포인트 차감
   - 주문, 배송, 결제 정보 저장
5. 주문 완료 시 주문 내역 반환

### [제약 조건]
- 로그인한 사용자만 주문할 수 있다. 
- 결제 시점에 상품 재고와 포인트 잔액이 충족되어야 한다. 

### [오류 처리]
- 로그인하지 않은 사용자가 요청할 경우, 401 Unauthorized 응답
- 존재하지 않는 상품을 주문하려는 경우, 404 Not Found 응답
- 주문 수량이 음수이거나 0인 경우, 400 Bad Request 응답
- 배송 정보가 없는 경우, 400 Bad Request 응답
- 상품 재고가 부족한 경우, 400 Bad Request 응답
- 사용자의 포인트 잔액이 부족한 경우, 400 Bad Request 응답
- 시스템 오류가 발생한 경우, 500 Internal Server Error 응답
